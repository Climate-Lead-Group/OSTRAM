1 What Are Git & GitHub?

Think of it this way

Git is like the “Track Changes” feature in Microsoft Word — but for every file in a project. It remembers every change anyone makes, when they made it, and why.

Git (the tool)

A free program that runs on your computer. It watches your project folder and records a complete history of every change you save.

GitHub (the website)

A cloud service (like Google Drive, but for code) where the team stores a shared copy of the project so everyone can stay in sync.

Key Vocabulary

TermWhat It MeansEveryday Analogy
Repository (repo)Your project folder, tracked by GitA shared Google Drive folder
CommitA saved snapshot of your changesPressing “Save” with a description
PushUploading your commits to GitHubSyncing your files to the cloud
PullDownloading the latest changes from GitHubRefreshing to see teammates’ updates
CloneDownloading the full project for the first timeMaking a copy of the shared folder
BranchA parallel copy of the project for experimentingMaking a draft copy before editing a report

2 Why We Need Version Control for OSTRAM

OSTRAM is a complex energy model with configuration files, Python scripts, and data pipelines that multiple team members work on simultaneously. Here is why Git is essential:

The Problem Without Git

File Chaos

Without Git, files end up named Config_FINAL.yaml, Config_FINAL_v2.yaml, Config_FINAL_v2_REAL.yaml. Nobody knows which version is current or who changed what.

Collaboration Conflicts

If two people edit Config_MOMF_T1_AB.yaml at the same time and email it around, someone’s changes will get overwritten and lost forever.

What Git Solves

BenefitHow It Helps OSTRAM
Complete HistoryEvery change to every file is recorded. You can see who modified B2_Executing_OG_Model.py last Tuesday and exactly what they changed.
Safe ExperimentationWant to try a new solver configuration? Create a branch, test it, and if it breaks things, just go back — the original is untouched.
Team SynchronizationEveryone always knows where the latest, working version lives. No more emailing files back and forth.
ReproducibilityOSTRAM uses DVC for data versioning. Git + DVC together mean you can reproduce any past model run exactly, which is critical for peer review and auditing.
Disaster RecoveryAccidentally deleted a file? Changed a config that broke the model? Git lets you restore any previous version in seconds.
Bottom Line

Git is not just for software developers. For a shared, evolving model like OSTRAM, it is the professional standard for keeping work safe, organized, and collaborative.

What Is the “Command Line”?

Command Prompt (cmd)

The basic Windows terminal. Open it via Start Menu → type cmd → press Enter. Use this for Git commands.

Anaconda Prompt

Comes with Miniconda/Anaconda. Pre-configured for Python and Conda. Open via Start Menu → search “Anaconda Prompt”. Use this for running OSTRAM.

Think of it this way

The Command Prompt is like texting your computer instead of clicking buttons. Every command in this guide will appear in a dark box — type exactly what you see and press Enter.

4 Creating a GitHub Account

  1. Go to GitHub
    Open your browser and visit: https://github.com
  2. Click “Sign up”
    Use your work email address. Choose a recognizable username (e.g. firstname-lastname-CLG).
  3. Verify your email
    GitHub will send a verification email. Click the link inside to activate your account.
  4. Ask to be added to the OSTRAM repository
    Send your GitHub username to the project administrator so they can grant access to Climate-Lead-Group/OSTRAM.
    Important

    You will not be able to push changes until the admin adds you as a collaborator. Accept the GitHub invitation email you receive.

5 First-Time Git Configuration

You only need to do this once on your computer. Open Command Prompt and run these commands with your own information:

# Tell Git your name (appears in the history)
git config --global user.name "Your Full Name"

# Tell Git your email (must match your GitHub account)
git config --global user.email "[email protected]"

Verify Your Configuration

git config --list
# Look for user.name and user.email in the output
Why does this matter?

Every commit in Git is stamped with your name and email — your digital signature on every change.

6 Getting the OSTRAM Repository

This downloads the entire OSTRAM project to your computer. You only do this once.

  1. Choose where to put the project
    Open Command Prompt and navigate to your target folder. For example, to place it on your Desktop: 
    cd %USERPROFILE%\Desktop
  2. Clone (download) the repository 
    git clone https://github.com/Climate-Lead-Group/OSTRAM.git

    This creates a folder called OSTRAM with all project files inside.

  3. Enter the project folder 
    cd OSTRAM
  4. Confirm everything is working 
    git status
# Should show: On branch main — nothing to commit, working tree clean
⚠️ Avoid Cloud-Synced Folders

Do not clone the OSTRAM repository into a folder that is actively synced by OneDrive, Google Drive, Dropbox, or any similar service. Sync clients continuously lock and rewrite files in the background, which causes access denied / read-write errors when Git or the solver tries to access the same files simultaneously. Clone to a local path instead — your Desktop (%USERPROFILE%\Desktop) or a dedicated local folder such as C:\Projects\OSTRAM are both safe choices.

First-Time Login

The first time you interact with GitHub from the terminal, a browser window will pop up asking you to log in. Sign in and authorize Git to access your account. You only need to do this once.

7 Installing Miniconda

Miniconda manages Python and all scientific libraries that OSTRAM needs. Think of it as an “app store” for Python tools.

  1. Download Miniconda
    Go to: Installing Miniconda — Anaconda Documentation
    Download the Windows 64-bit installer (.exe file).
    Miniconda vs. Anaconda

    If you already have Anaconda installed, skip this section — they serve the same purpose. Anaconda simply includes more pre-installed packages.

  2. Run the installer
    Double-click the downloaded file. During installation:
    • Accept the license agreement
    • Choose “Just Me” for the installation scope
    • Keep the default installation folder
    • Check the box: “Add Miniconda3 to my PATH environment variable” (ignore the installer warning — this is fine)
  3. Verify the installation
    Close and reopen Command Prompt, then type: 
    conda --version
# You should see something like: conda 24.7.1
Using the Anaconda Prompt

If conda does not work in regular Command Prompt, use Anaconda Prompt instead: Start Menu → search “Anaconda Prompt” → open it. All subsequent steps should be performed in Anaconda Prompt.

8 Running OSTRAM for the First Time

With Miniconda installed and the repository cloned, you can launch OSTRAM directly. The run.py script handles environment creation and dependency installation automatically on first launch.

  1. Open Anaconda Prompt
    Start Menu → search “Anaconda Prompt” → open it. All OSTRAM commands should be run from Anaconda Prompt, not regular Command Prompt.
  2. Navigate to the OSTRAM folder 
    cd path\to\OSTRAM
# Example: cd %USERPROFILE%\Desktop\OSTRAM
  3. Run OSTRAM 
    python run.py
# On first run, this automatically creates the Conda environment
# and installs all required dependencies including GLPK and CBC.
    What happens on first run?

    run.py reads environment.yaml, creates the OSTRAM-env Conda environment, and installs all required packages including the GLPK and CBC solvers. This may take several minutes on first launch.

Subsequent runs

Every time you open a new Anaconda Prompt to work with OSTRAM, activate the environment first: conda activate OSTRAM_env. Without this step you will see errors about missing packages.

9 Optimization Solvers

OSTRAM uses the OSeMOSYS energy model, which requires a linear programming solver to find optimal solutions.

Think of it this way

If OSTRAM is the recipe, the solver is the oven. You can prepare all the ingredients (data, configuration), but without the oven (solver) you cannot bake the result (run the optimization).

Required vs. Optional Solvers

SolverStatusLicenseNotes
GLPKAuto-installedFree & open sourceDefault solver. Installed automatically by run.py on first launch.
CBCAuto-installedFree & open sourceFaster alternative for larger runs. Also installed automatically.
CPLEXOptionalCommercial (free academic)Significantly faster for large-scale models. Free for academics via IBM Academic Initiative.
GurobiOptionalCommercial (free academic)High-performance solver. Free academic licenses at gurobi.com/academia.
Which solver should I use?

GLPK and CBC are installed automatically and are sufficient for most OSTRAM runs. If you are working with large multi-region scenarios and run times are excessive, consider requesting an academic license for CPLEX or Gurobi.

Installing Optional Solvers (after activating env)

Activate the OSTRAM environment first, then install the optional solver of your choice:

# Step 1 — activate the OSTRAM environment
conda activate OSTRAM-env

CPLEX (Optional)

Download from IBM Academic Initiative. After installing the full CPLEX suite, add the Python bindings:

pip install cplex

Gurobi (Optional)

Download from Gurobi Academia. Then add the Python bindings:

conda install -c gurobi gurobi

10 Setting Up the OSTRAM Environment (Manual)

The run.py script handles environment setup automatically. This section describes the manual steps for reference or troubleshooting.

Option A: Automatic Setup (Recommended)

Open Anaconda Prompt, navigate to the OSTRAM folder, and run:

cd path\to\OSTRAM
python run.py
# Automatically creates the Conda environment and installs all dependencies.

Option B: Manual Setup

# Create the environment from OSTRAM's configuration file
conda env create -f environment.yaml

# Activate the environment
conda activate OSTRAM_env

# Verify Python version
python --version
# Should show Python 3.10.x
What is a Conda environment?

A self-contained toolbox. OSTRAM needs specific versions of Python libraries. The environment ensures those exact versions are installed without conflicting with other programs on your computer.

Remember

Every time you open a new Anaconda Prompt to work with OSTRAM, activate the environment first: conda activate OSTRAM_env. Without it, you will see errors about missing packages.
